Pack format reference

    
Each Minecraft Java version expects a specific pack_format. Click any card to use it.

  

  

  

    Since snapshot 25w31a (1.21.9+) pack_format was replaced by min_format / max_format.

How a data pack works

    
Anatomy of the file tree this tool generates for you.

  

  

    
pack.mcmeta
Root manifest. Declares the pack description and the pack_format.

    
data/<namespace>/
Your unique folder so packs don't collide.

    
function/
.mcfunction files — one command per line. Run via /function ns:name.

    
recipe/
JSON recipes auto-registered when the pack loads.

    
loot_table/
JSON tables deciding what drops from blocks, mobs, chests.

    
advancement/
JSON achievements with triggers, criteria, and rewards.

    
predicate/
Reusable boolean conditions — random chance, weather, time, scores.

    
item_modifier/
Reusable item transforms applied with /item modify.

    
damage_type/
Custom damage sources (added in 1.19.4).

    
tags/function/
load.json runs once on load. tick.json runs every game tick.

    
tags/block/ · tags/item/
Group blocks or items under one ID — referenced as #ns:name.

    
tags/entity_type/ · tags/fluid/
Group entities or fluids for use in predicates and commands.

    
structure/
.nbt files for /place template and Jigsaw blocks.

    
worldgen/
Biomes, features, noise settings, structures, world presets.

    
dimension/ · dimension_type/
Define custom dimensions and their physics.

    
enchantment/ · trim_pattern/
Data-driven enchantments (1.21+) and armor trims (1.20+).
